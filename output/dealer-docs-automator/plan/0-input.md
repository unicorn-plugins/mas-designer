## 주제

**개통·서류 자동 처리 (Activation Automation — Dealer Document Pipeline)**

신규 가입·번호이동·기기변경 등 개통 업무에서 발생하는 서류 수집·검증·작성·본사 제출 전 과정을
AI 멀티에이전트 시스템으로 자동화함.

### 범위 (In-Scope)
- 신분증·가족관계증명서·법인서류 등 **OCR 기반 정보 추출**
- 명의자 일치·약정 위반·미성년자 동의 등 **규정 사전 검증(컴플라이언스)**
- 개통 서류(위임장·약관·가입신청서) **자동 작성**
- **본사 개통 시스템(BSS) 제출** 및 반려 사유 피드백 루프
- **사기·명의도용 위험 탐지**(경찰청 DB 연계·이상 패턴)
- 영업사원용 **개통 진행 현황·체크리스트** 생성

### 범위 외 (Out-of-Scope)
- 요금제 추천·가입자 유치 코치 (옵션 A)
- 해지 방어·리텐션 (옵션 C)
- 개통 후 요금 청구·AS·민원 처리

## 현재 상황 · 고통 지점 (도메인 지식 기반 자동 발굴)

### 정량 지표 (업계 통상치)

| 지표 | 현재 수준 | 의미 |
|------|----------|------|
| 개통 서류 반려율 | **15~25%** | 본사 BSS가 서류 오류·누락·규정 위반으로 재제출 요구 |
| 건당 서류 작성 시간 | 평균 **25~40분** | 신분증 스캔·동의서 출력·서명·규정 체크리스트 확인 |
| 영업사원 1일 개통 건수 | **5~8건** (번화가 대리점), **2~4건** (외곽) | 서류 처리 시간이 실질적 병목 |
| 명의도용·사기 개통 발생률 | **0.3~1.2%** | 1건당 평균 손실 100~300만 원 + 법적 리스크 |
| 개통 후 7일 내 철회율 | **3~6%** | 불완전판매·오해·규정 미준수로 인한 철회 |
| 반복 질문 대응 시간 | 영업사원 근무의 **15~20%** | "이 서류 이거 써도 되나" "약정 위반인가" 등 본사 CS 문의 |

### 질적 고통 지점

#### 1. 서류 품질·정확도 문제
- 신입 영업사원의 **규정 이해 편차** → 기재 오류·누락
- 가족관계증명서의 **계속 변하는 포맷**(최근 3년간 3회 변경) 적응 지연
- 법인 개통의 경우 **사업자등록증·위임장·재직증명** 다종 서류가 필요하나 체크리스트가 없거나 오래됨
- 본사가 요구하는 **서명 위치·필수 기재항목**이 SKU별로 다름

#### 2. 컴플라이언스 리스크
- **명의도용**: 최근 1인 명의 대량 개통, 허위 신분증(위조·도용), 미성년자 명의 오남용 등 사기 수법 다양화
- **약정 위반**: 기존 약정 미해지 상태에서 번호이동, 결합 해지 페널티 미인지
- **불완전판매**: 단말 지원금·공시지원금·선택약정 비교 설명 누락
- **KYC·AML 규제**: 2023년 이후 통신 가입도 자금세탁방지법 대상 확대, 위험 고객 스코어링 의무
- **개인정보보호법**: 고객 신분증 원본 **부적절 보관·전송**이 주된 과태료 사유

#### 3. 업무 효율·경험 문제
- **OCR 미도입** 또는 도입해도 정확도 70% 수준 → 수동 재입력 반복
- **본사 BSS 시스템** 응답 느림(피크타임 5~10초) + 반려 사유 메시지가 추상적 → 영업사원이 원인 파악 어려움
- 대리점 내 **CS 콜센터 의존**: 본사 영업지원에 전화해 "이 서류 이거 맞냐" 반복 문의, 대기 시간 5~15분
- **야간·주말 개통 지원 부재**: 본사 영업지원 영업시간 외 개통 중단

#### 4. 관리·추적 문제
- **개통 진행 단계가 불투명**: 고객이 "내 개통 언제 되냐" 문의 시 영업사원도 본사 시스템 재조회
- 반려 사유가 **축적 데이터로 학습되지 않음** → 동일 실수 반복
- 대리점 간 **베스트 프랙티스 공유 부재**

### 근본 원인 가설 (problem-analyst가 5WHY로 심화 예정)

1. 왜 반려율이 높은가? → 서류 품질이 들쑥날쑥 → 왜? → 규정이 복잡하고 자주 변함 + 체크가 사람 기억 의존
2. 왜 사기 개통이 발생하는가? → 위조·명의도용을 현장에서 감지할 수단 부재 → 왜? → 본사 사기 탐지 모델이 **실시간 피드백**을 현장에 주지 못함
3. 왜 건당 30분이 걸리는가? → 수기 입력·출력·서명·스캔 반복 → 왜? → 본사 시스템과 대리점 PoS의 **데이터 흐름 단절**

## 대상 사용자 (Primary User)

**대리점 영업사원**

- 연령대 20~50대, 업무 경력 편차 큼(1년 미만 ~ 20년+)
- 디지털 활용도 중간(스마트폰·PC 기본, AI 도구 경험은 적음)
- 성과 평가: **개통 건수 + 반려·철회율 + 사기 방어**
- 핵심 가치: **시간 절약 + 실수 방지 + 본사 호출 감소**
- UI 선호: 한국어 GUI, 태블릿/PC 공용, 큰 글씨, 단계별 체크리스트

### 2차 사용자 (이해관계자)

- **대리점 점장**: 일 단위 대시보드·이상 징후 알림 필요
- **본사 영업지원팀**: 대리점 문의 감소, 규정 위반 사전 차단, 사기 방어 성공
- **본사 컴플라이언스·리스크 부서**: 개통별 위험 스코어, 감사 로그 추적

## 보고 대상 (Executive Audience)

**CIO (Chief Information Officer)**

### CIO가 듣고 싶어하는 것
1. **리스크 감축 효과** — 사기 개통 손실 감소, 규제 과태료 예방 금액
2. **반려율 개선 → BSS 재처리 비용 절감** — 본사 운영비 감소
3. **영업사원 생산성 향상** — 개통 처리량 증가 → 매출 기여
4. **기술 거버넌스** — 데이터 보안·모델 운영·벤더 종속성 관리 방안
5. **확장 로드맵** — 파일럿 → 전 대리점 확산 → 옴니채널 연계 그림

### PPT에 반드시 포함될 메시지 (GREAT 2 WHY)
- **고객 WHY (영업사원·점장 관점)**: "서류 작성이 반으로 줄고, 본사에 묻지 않아도 95% 처리됨"
- **기업 WHY (본사·CIO 관점)**: "반려율과 사기율이 동시에 감소하여 운영비·리스크 이중 절감, ROI 18개월 내 회수"

## 제약 · 비기능 요구사항

### 필수
- **PII 마스킹**: 신분증·주민등록번호·주소·전화번호 등은 로그·저장소·프롬프트에서 마스킹
- **개인정보보호법 · 정보통신망법 · 전자서명법** 준수
- **본사 BSS 연동**: 실시간 API 호출, 장애 시 큐잉·재시도
- **KYC·AML 규제** 수용: 위험 스코어링 결과를 감사 로그로 저장

### 권장
- **엣지 추론 / 하이브리드**: OCR·PII 마스킹은 대리점 로컬에서, 복잡 추론은 본사 클라우드
- **오프라인 폴백**: 본사 BSS 장애 시 큐에 대기 후 복구 시 자동 제출
- **다국어**: 외국인 가입자 대비 영어·중국어 본인확인 지원
- **감사 추적**: 모든 에이전트 판단·사용자 확인 단계 로깅

### 선호 스택 (시사항목)
- **멀티모달 LLM**: Gemini / GPT-4o / Claude (이미지 OCR + 추론)
- **RAG**: 규정집·상품 카탈로그·본사 공지 통합 인덱스
- **LangGraph**: 에이전트 토폴로지, Retry·Escalation 분기
- **MCP**: 본사 BSS·KYC·경찰청 DB 연동
- **Vector DB**: 반려 사례·사기 패턴 학습용
