### 한 문장 요약

> **"통신 대리점 개통 접수 순간, 사기·명의도용·규정 위반을 병렬 에이전트가 2초 내 판정하고,

  PII    Face KYC/AML  이상      서류 생성     감사 로그

    │    │    │        │

    [본사 BSS 제출 (Green) 또는 에스컬레이션 (Yellow/Red)]

### 구성 에이전트 (5~7개)

| 에이전트 | 역할 | 병렬성 |
|---------|------|:------:|
| **Orchestrator** | LangGraph 상태 관리, 라우팅, Human-in-Loop | - |
| **ID Verifier** | 신분증 OCR + VLM 위변조 + 얼굴·라이브니스 | ✅ |
| **Compliance RegGuard** | 규정 RAG, 조항 인용, 약정·미성년 검증 | ✅ |
| **Fraud Scorer** | 다회선·블랙리스트·패턴 ML 스코어링 | ✅ |
| **Document Generator** | 위임장·가입신청서·동의서 템플릿 기입 | 순차 (검증 후) |
| **Audit Logger** | 전체 이벤트·판정·사용자 확인 구조화 저장 | 병렬 모니터 |
| **Dealer Copilot** (선택) | 영업사원에게 설명 스크립트 제공 | 필요 시 |

## 선정 근거 5항목

### 1. 문제 정합성 (킹핀 직결)
킹핀 R2(위험 신호가 개통 순간 현장 도달 안 함)를 **직접** 해결. 실시간 2초 스코어링이 핵심 요건을 충족.

### 2. B/F 평가 결과
B=5, F=4 (합 9) — **No Brainers** 사분면 최상단. RegGuard(B4F5)도 동률이지만 임팩트에서 RTA 우위.

### 3. 우선순위 매트릭스 위치
No Brainers + 전략 사분면. 다른 후보(IDV+·RegGuard·AutoDoc)는 **RTA의 내부 에이전트로 흡수** 가능하므로 RTA가 전사 아키텍처의 "뿌리" 역할.

### 4. AI 실현 가능성 4요소
- 데이터 가용성 4/5 — 본사·외부 DB 연계 가능
- 모델 적합성 5/5 — LangGraph + Flash/Haiku 추론으로 2초 내 달성
- 인프라 준비도 3/5 — MCP 서버 구축·거버넌스 합의 필요 (핵심 과제)
- 운영 지속성 4/5 — FP 피드백 루프·설명가능성 기반
- 소계 **16/20** (파일럿 가능 수준)

### 5. 방향성(3대 축) 정합성
- **Real-time Risk Triage** — RTA의 핵심 기능 자체
- **Explainable Compliance** — 규정 조항 인용 + 판정 근거 UI로 내장
- **Progressive Intelligence Standard** — RTA 아키텍처가 타 후보를 흡수·확장하며 **표준 지능화**
