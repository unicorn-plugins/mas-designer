## 1. SCAMPER 전개

### S — Substitute (대체)
**S1. 영업사원 육안 판단 → AI OCR+VLM(비전 언어 모델) 판단 대체**  
신분증·가족관계증명 위변조 탐지를 사람 육안에서 **멀티모달 LLM의 위변조 특징 탐지**로 대체.

**S2. 본사 사후 배치 사기 탐지 → 현장 실시간 스코어링 에이전트 대체**  
배치형 부정 탐지 모델을 개통 순간 **실시간 리스크 에이전트**로 대체.

### C — Combine (결합)
**S3. OCR + KYC 조회 + 약정 이력 + 경찰청 분실 신고 DB를 단일 리스크 스코어로 통합**  
개별 검증을 **LangGraph 병렬 노드 + 스코어 합산**으로 통합.

**S4. 규정 RAG + 사기 탐지 에이전트 결합**  
동일 체인에서 **규정 위반 + 사기 의심** 두 축을 동시 판정.

### A — Adapt (각색)
**S5. 은행 여신 심사의 머신러닝 스코어링 + 설명 가능성(XAI) 기법을 개통에 각색**  
여신 거절 사유 제시 방식을 사기 개통 차단에 적용.

**S6. 관세·세관 X-ray AI 탐지의 멀티모달 검사 패턴을 신분증·서류 이미지 검사에 각색**  
다각도 이미지 특징 검출 패턴 이식.

### M — Modify (변형)
**S7. 리스크 스코어를 단일 숫자 → Green/Yellow/Red 3단계 + 근거 리스트로 변형**  
영업사원이 즉시 행동할 수 있는 형태로 가공.

**S8. 개통 트랜잭션을 단계형 → 병렬 파이프라인으로 변형**  
OCR·KYC·규정·서류 생성을 병렬 수행.

### P — Put to Other Use (다른 용도 활용)
**S9. 리스크 스코어링 아키텍처를 해지방어·리텐션 재사용**  
사기·이탈 공통 이상 탐지 인프라로 확장.

**S10. 개통 감사 로그를 점장 대시보드·본사 컴플라이언스 리포트에 재사용**  
단일 로그 → 다목적 활용.

### E — Eliminate (제거)
**S11. 본사 영업지원 콜 문의 제거 — 규정 RAG 에이전트가 현장 답변**

**S12. 수동 서류 출력·서명·재스캔 제거 — 전자서명·태블릿 서명 패드 통합**

### R — Reverse / Rearrange (역발상 / 재배열)
**S13. "개통 후 사후 검증" → "개통 전 사전 차단"으로 순서 뒤집기**  
본사 BSS 제출 전 대리점 단말에서 거부 결정.

**S14. 사람 중심 워크플로우 → 에이전트 중심 워크플로우로 역전**  
영업사원은 **"고객과의 대화"**에만 집중, 서류는 에이전트가 주도.

## 2. Steal & Synthesize (타 도메인 이식)

### R1. 금융권 — 카드 부정결제 실시간 탐지
- **원천**: 시장 선도 카드사의 실시간 FDS(Fraud Detection System)는 **50~200ms 스코어링** + 이상 시 **홀드·콜백**
- **이식**: 개통 건을 카드 거래처럼 취급, 2초 내 스코어링 + Yellow 시 점장/본사 에스컬레이션
- **S15. FDS-Inspired Activation Fraud Detection**

### R2. 공항 보안 — CT(입체) 영상 + AI 위험 분류
- **원천**: 공항 수하물 스크리닝에서 **멀티앵글 이미지 + 3D 딥러닝 + 수동 재검사 큐**
- **이식**: 신분증을 **다각도 촬영 + VLM + 의심 시 영상 통화 본인확인 큐**로 이식
- **S16. Airport-Inspired Multi-View ID Verification**

### R3. 법률 리걸테크 — 판례 RAG + 인용 표시
- **원천**: AI 법률 리서치 도구가 **판례 문단 인용 + 신뢰도 표시**로 변호사 의사결정 보조
- **이식**: 규정 RAG가 **규정 조항 직접 인용 + 해당 사유 표시**로 영업사원 판단 보조
- **S17. LegalTech-Inspired Regulation Citation**

### R4. 의료 — 의사 결정 지원 시스템 (CDSS)
- **원천**: 의사에게 증상 기반 차별진단 제공 + 근거 논문 링크 + 최종 판단은 의사
- **이식**: 영업사원에게 위험 시나리오 + 근거 + 최종 결정권 유지
- **S18. CDSS-Inspired Decision Support for Agents**

### R5. 신원증명 업계 (IDV)
- **원천**: Onfido, Jumio 등의 **셀피 + 신분증 + 액티브 라이브니스 체크**
- **이식**: 고객 얼굴 셀피 + 신분증 사진 얼굴 비교 + 움직임 감지를 현장 태블릿으로
- **S19. IDV-Inspired Selfie Matching**

## 3. AI 패턴 카드 (ai-pattern-lib 세부역할 제공)

### P1. 분류 (Classification)
**S20. 위험도 분류기** — 개통 요청을 Low/Mid/High 리스크로 분류

### P2. 추출 (Extraction / OCR + NER)
**S21. 신분증 OCR + 주요 필드 추출** — 주민번호(마스킹), 이름, 주소, 발급일

### P3. 검증 (Verification)
**S22. 얼굴 대조 + 라이브니스** — 멀티모달 VLM

### P4. 추론 (Reasoning)
**S23. 근거 기반 규정 해석 에이전트** — ReAct 패턴, 규정 RAG로 판례/조항 인용

### P5. 생성 (Generation)
**S24. 서류 자동 생성** — 위임장·동의서·가입신청서 템플릿 + 필드 채움

### P6. 에이전트화 (Agentification)
**S25. 다중 에이전트 병렬 오케스트레이션** — LangGraph 기반 OCR/KYC/규정/서류/사기 탐지 동시 실행

### P7. 예측 (Prediction)
**S26. 철회 가능성 예측** — 고객 프로파일 + 상담 맥락으로 7일 내 철회 확률 예측

### P8. 검색 (Retrieval / RAG)
**S27. 규정·상품·경쟁사 비교 RAG** — 실시간 질의응답

### P9. 요약 (Summarization)
**S28. 개통 건별 감사 로그 자동 요약** — 점장·본사 리포트용

### P10. 다이얼로그 / 스크립트 (Dialogue)
**S29. 영업사원 설명 스크립트 생성** — 약관·혜택 설명을 고객 맞춤으로
