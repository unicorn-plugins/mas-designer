# 6. 멀티모달 I/O 설계

> 입력(텍스트·이미지·음성·문서) 및 출력(리포트·시각자료·음성) 포맷·모델·제약.
> 근거: `references/01-multimodal-ai.md` §2~§5

---

## 입력 모달리티

### 1) 이미지 (신분증·서류·얼굴)

| 소스 | 포맷 | 해상도 | 처리 에이전트 | 모델 |
|------|------|:------:|--------------|------|
| 신분증 (주민등록증·운전면허) | JPEG/PNG | 최소 1280×720, 권장 1920×1080 | IDV+ | Gemini 2.5 Pro Vision (주력) + Claude Sonnet Vision (백업) |
| 외국인등록증·여권 | JPEG/PNG | 최소 1280×720 | IDV+ | 동일 |
| 가족관계증명서·법인서류 | PDF·JPEG | 300dpi 이상 | Doc Generator + IDV+ | VLM + OCR 결합 |
| 얼굴 셀피 | JPEG | 최소 512×512, 30fps 짧은 영상 가능 | IDV+ | ArcFace + Liveness Library |

### 전처리 공통
- 자동 회전·원근 보정
- **PII 마스킹 미리보기** (뒤 6자리 주민번호 등) → 영업사원 UI에는 마스킹된 버전만
- 원본은 **로컬 암호화 저장소**에, 프롬프트에는 마스킹 버전 전송

근거: [1]

### 위변조 탐지 특징
- **합성 탐지**: 픽셀 연속성·노이즈 패턴 (Copy-Paste 탐지)
- **재촬영 탐지**: 반사·모이레 패턴 (화면을 다시 찍은 경우)
- **변조 탐지**: 글자·홀로그램 영역 어색함 (Generative AI로 수정된 경우)

근거: [2]

---

### 2) 텍스트 (영업사원 입력·고객 대화)

| 소스 | 용도 | 처리 에이전트 | 모델 |
|------|------|--------------|------|
| 영업사원 질의 | Dealer Copilot 응답 | A7 Dealer Copilot | Claude Sonnet / Gemini Flash |
| 고객 발화 기록 (선택적) | 불완전판매 감지 | Audit Logger | LLM 요약 + 이상 탐지 |

---

### 3) 음성 (선택 - 전화 상담 녹취)

| 소스 | 용도 | 모델 |
|------|------|------|
| 콜센터 녹취 (본사→대리점 문의) | Dealer Copilot 학습 데이터 | STT → 요약 → FAQ 인덱스 |
| 외국인 고객 다국어 대화 | 자막 실시간 번역 (NFR-07) | Gemini Live Audio 또는 Whisper + 번역 LLM |

**출력 우선 설계**: 현 단계는 입력 위주, 음성 출력(TTS)은 후속 확장.

---

### 4) 문서 (PDF)

| 문서 유형 | 처리 |
|----------|------|
| 본사 규정 PDF (인덱싱) | 텍스트 추출 → 청킹 (Section 5 참조) |
| 서류 원본 PDF | OCR + 필드 추출 (VLM) |

## 출력 모달리티

### 1) 영업사원 UI (시각)

| 요소 | 포맷 | 특징 |
|------|------|------|
| 위험 스코어 배지 | Green/Yellow/Red 원형 + 점수 | 2초 내 렌더링 |
| 근거 리스트 | 카드 그리드 (3~5건) | 조항 ID 클릭 시 원문 팝업 |
| 위변조 하이라이트 | 이미지 위 bbox 오버레이 | 의심 영역 붉은 테두리 |
| 진행 단계 바 | 5단계 프로그레시브 바 | 단계별 아이콘 + 예상 남은 시간 |

### 2) 문서 생성 (PDF)

- **가입신청서·위임장·동의서**: pptx·docx가 아닌 **PDF/A-2b** (장기 보관 표준)
- **법인 특화 템플릿**: 대표자·담당자 2단 서명
- **무결성 해시**: 생성 시 SHA-256, 본사 BSS에 메타데이터로 동봉

### 3) 감사 보고서 (정기)

| 대상 | 주기 | 포맷 |
|------|:----:|------|
| 점장 일일 대시보드 | 일 | HTML + Chart.js (9차트: 건수·반려율·Red·FP 등) |
| 본사 컴플라이언스 월간 | 월 | PDF + 요약 LLM 생성 |
| 외부 감사 (분기) | 분기 | PDF + 이벤트 원본 로그 첨부 |

### 4) 경영진 PPT (STEP 3 산출)

- **GREAT 2 WHY 중심**: 슬라이드 패턴(A~F) 기반
- **이미지 일러스트**: `generate_image` (Gemini Nano Banana) 자동 생성
- **상세**: STEP 3에서 pptx-spec-writer가 명세 + `generate-pptx` 스킬이 빌드

## 모델 선정 기준

| 요건 | 우선 모델 | 근거 |
|------|----------|------|
| 2초 내 다중 모달 판정 (Orchestrator 종합) | **Claude Sonnet 4.6** or **Gemini 2.5 Flash** | 지연 우선 |
| 고정확 VLM (위변조·법인서류) | **Gemini 2.5 Pro** | VLM 품질 우위 |
| 얼굴 대조 | **ArcFace (로컬)** | 온프레미스 프라이버시 |
| 라이브니스 | **로컬 Active Challenge** | 오프라인 가능 |
| 한국어 RAG 임베딩 | **text-embedding-3-large** or **KoSimCSE** | 다국어 vs 국내 리전 |
| 이미지 생성 (PPT용) | **Gemini Nano Banana** | 한국어 프롬프트 지원 |

## 멀티모달 파이프라인 예시 (신분증 검증)

```
[신분증 촬영 (태블릿)]
      │
      ▼
[로컬 전처리]
      ├─ 회전·원근 보정 (OpenCV)
      ├─ 품질 점수 (해상도·초점)
      └─ PII 영역 블러 옵션 (UI 표시용만)
      │
      ▼
[OCR: Gemini Pro Vision (Cloud, 국내 리전)]
      │   → 필드 추출 (이름, 생년월일, 발급일)
      │
      ▼
[VLM 위변조 탐지]
      │   → 위변조 스코어 + 의심 영역 bbox
      │
      ▼
[얼굴 크롭 → ArcFace 임베딩 (로컬)]
      │
      ▼
[고객 셀피 임베딩 (로컬)]
      │
      ▼
[임베딩 유사도 비교 (로컬)]
      │
      ▼
[라이브니스 체크 (로컬 Active Challenge)]
      │
      ▼
[IDV 결과 State 반영]
```

**프라이버시 원칙**: 원본 신분증·셀피는 **로컬 실행** 단계 밖으로 나가지 않음. VLM은 클라우드 호출하나 **PII 영역 마스킹 후 전송**. 근거: [1], [4]
